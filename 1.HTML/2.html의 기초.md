## 2.3 Emphasis
* 내용을 좀 더 특별하게 강조
  * ```<em>강조</em>```
  * ```<strong>강조</strong>```

* ```<br>``` : break. 줄바꿈. 남용 금지   
+```<br>과 <br/>``` : 각각 HTML5와 XHTML에서 선호. 혼용되는 것도 있음

## 2.4 Anchor
* ```<a></a>``` : 현위치에서 다른 위치로 이동  
* href : hypertext reference (html문서 웹페이지의 주소값). 필수 속성! 
>마크업에는 정답이 없음. 내가 생각하는 논리에 적합한 대로!


### href값 설정 방법
* 절대 경로 : url주소 입력
* 상대 경로 : 폴더 경로 입력
* 페이지 내 이동 : 이동하고자하는 부분의 id값을 적기
```
<a ref="#hello">
```
* 메일쓰기 : 메일 주소 클릭시 메일 쓰기 연결
```
<a ref="mailto:메일주소">
```
* 전화걸기 : 사이트에서 전화번호 누르면 바로 통화연결
```
<a ref="tel:01012345678">
```
### target 속성
* ```target="_blank"``` : 새탭 열기    
+마크업 할 때 제일 먼저 해야하는 것 : 덩어리로 나누기

## 2.6 Lists
* ul : unordered list
* ol : ordered list
```
<ul>
 <li>항목</li>
 <li>항목</li>
</ul>
```
`<ul>`과 `<ol>`의 자식요소는 무조건 `<li>`만 가능  
+자식요소, 자손요소

```
<ol>
 <a>
  <li>녕녕</li>
 </a>
</ol>

<ol>
 <li>
  <a>녕녕</a>
 </li>
</ol>
```
둘 중 맞는 것은 아래 것.  
`<ol>`의 자식요소는 `<li>`임

>마크업이란 브라우저에게 어떤 정보를 가졌는 지 알려주는 것  
 어떻게 표시되든 중요하지 않음.
 순서가 중요하지 않은 리스트라는 걸 알려줬다는 게 중요함.
 
## 2.7 Description List
* 의미
  * 용어 정의시 사용하는 리스트
  * key-value 정보 정의시 사용하는 리스트
{key : value}  
{이름 : 녕녕}  
{직업 : 프론트엔드 개발자}  

* 태그
  * `<dl>` : 정의리스트 선언
  * `<dt>` : description term, key값에 해당
  * `<dfn>` : definition
  * `<dd>` : description date, value값에 해당  
 `<dl>`안에는 `<dt>,<dfn>`,`<dd>`가 반드시 따라오고, `<div>`까지 세 가지만 쓸 수 있음

* 태그 주의사항
  * good example
 ```
 <dl>
  <dt>녕녕</dt>
  <dd>프론트엔드 개발자</dd>
 </dl>
 ```
 ```
 <dl>
  <dt>
   <dfn>녕녕</dfn>
  </dt>
  <dd>1.프론트엔드 개발자</dd>
  <dd>2.영화를 조아해 ~~</dd>
  <dd>3.채소를 조아해 ~~</dd>
 </dl>
 ```
 ```
 <dl>
  <dt>
   <dfn>녕녕</dfn>
   <dfn>eun0leee</dfn>
  </dt>
  <dd>프론트엔드 개발자</dd>
 </dl>
 ```
 ```
 <dl>
  <dt>녕녕</dt>
  <dd>프론트엔드 개발자</dd>
  <dd><a href="https://github.com/eun0leee">녕녕 블로그</a></dd>
 </dl>
 ```
 ```
 <dl>
  <dt>녕녕</dt>
  <dfn>eun0leee</dfn>
  <dd>프론트엔드 개발자</dd>
  <dd><a href="https://github.com/eun0leee">녕녕 블로그</a></dd>
 </dl>
 ```
 ```
 <dl>
  <div>
   <dt>녕녕</dt>
   <dd>프론트엔드 개발자</dd>
  </div>
 </dl>
 ```
 ```
 <dl>
  <div>
   <dt>녕녕</dt>
   <dd>프론트엔드 개발자</dd>
  </div>
  <div>
   <dt>영화</dt>
   <dd>녕녕이가 좋아하는 거</dd>
  </div>
 </dl>
 ```

  * bad example
 ```
 <dl>
  <dt>녕녕</dt>
  <dd>프론트엔드 개발자</dd>
  <dt>영화<dt>
 </dl>
 ```
 `<dt>`와`<dd>`가 짝으로 따라와야함
 ```
 <dl>
  <section>
   <dt>녕녕</dt>
   <dd>프론트엔드 개발자</dd>
  </section>
 </dl>
 ```
 `<div>`만 가능
 ```
  <dt>녕녕</dt>
 ```
  단독 사용 불가. `<dl>`의 자식요소로만 존재
